# 我的主張

在讀書會中導入結構化筆記模板（統一的段落標題與填寫欄位），能讓討論從「分享感想」升級為「交叉比對觀點」。但前提是模板夠簡單——超過五個欄位，參與者就會開始敷衍填寫。

# 實踐經驗

從一月開始，我在每週四的讀書會試了三輪不同的筆記模板：

**第一輪（1 月）**：完整模板，含摘要、關鍵論點、疑問、個人經驗、行動計畫五個欄位。結果只有兩個人持續填寫，其他人覺得太像作業。

**第二輪（2 月初）**：精簡成三個欄位——「一句話重點」、「一個疑問」、「一個連結」（跟自己過去經驗的連結）。完成率從 30% 跳到 85%。

**第三輪（2 月中）**：沿用三欄模板，但在討論時投影所有人的筆記。效果最好——大家開始對比彼此的「疑問」欄，發現同一段話在不同人眼裡看到完全不同的東西。

意外發現：「一個連結」這個欄位效果最強。它迫使每個人把新知識和自己的經驗拉在一起，而不只是被動接收。

# 未解決的問題

- 三欄模板在 5-8 人的小組很有效，超過 10 人時投影對比就變得混亂。大型讀書會該怎麼處理？
- 目前只試過非虛構類書籍。如果讀的是小說或詩集，結構化筆記還適用嗎？
- 有些成員反映「一句話重點」反而最難寫，因為要把整章濃縮成一句話需要更高的理解力。這到底是 feature 還是 bug？

# 跟外部研究的對照

- Cornell 筆記法的研究顯示，結構化格式確實能提升回憶率（Pauk & Owens, 2010），但那些研究都是個人使用，沒有測試群組情境
- 《Make It Stick》提到「生成效應」——主動產出內容比被動閱讀更有效——我的「一個連結」欄位可能正好觸發了這個機制
- 不過，Roam Research 社群的經驗分享中有人提到，過度結構化反而會抑制創意聯想。我的第一輪失敗可能不只是「欄位太多」，而是結構本身框住了思考方向
